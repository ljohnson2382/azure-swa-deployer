# SOP: Azure Static Web Apps IaC (azure-swa-deployer)

This folder contains the Standard Operating Procedure (SOP) for using and maintaining the Azure Static Web Apps infrastructure-as-code (IaC) framework in this repository.

Use this SOP when onboarding a new client, running local validation, or managing the CI/CD pipeline for infrastructure. The repo itself is intentionally configured to not perform deployments; see "Disablement" for details.

## Contents

- 01-Prerequisites.md — System and account requirements
- 02-Quickstart.md — Fast path to validate and run locally
- 03-Deployment-Procedures.md — Step-by-step deployment instructions (for a separate deploy repo)
- 04-Disable-Repo-Deployment.md — How and why this repository is disabled for deployments
- 05-Rollback-and-Recovery.md — Rollback and recovery procedures
- 06-Security-and-Secrets.md — Secrets management and principle of least privilege
- 07-Troubleshooting.md — Common problems and resolutions
- 08-Change-Log.md — SOP change log and history

## Contact
For questions about this SOP or the framework, open an issue in the repository or contact the infrastructure owner listed in your client onboarding documents.
